ComforTroll 是一个面向环境设计研究与户外现场调查的 Android 多传感器采集工具。它可以同步记录位置、环境光、姿态、运动状态和设备运行信息，适用于热舒适巡测、田野调查与空间环境调查。

## 它能做什么

- 按次保存巡测任务，旧记录不会被新实验“温柔覆盖”。
- 记录地点、人员、真实时间戳、定位与设备传感器数据。
- 根据当前手机的硬件能力动态识别传感器；有就显示，没有就不假装有。
- 自由选择需要连续记录的传感器，减少无意义的数据洪水。
- 实时显示定位、GNSS、照度、加速度、陀螺仪、电池与运行状态。
- 点击实时指标查看对应折线变化，现场判断数据到底在工作还是在表演。
- 添加预设或自定义现场事件，给后期分析留下一点人类记忆。
- 生成巡测自检与质量报告；发现异常后仍可由用户确认开始。
- 按单次任务导出 CSV，也可完整导出原始数据、事件和质量信息。
- 提供多套主题，包括不想让屏幕像调色盘爆炸的白灰主题。

## 四个主要模块

| 模块 | 用途 |
|---|---|
| 首页 | 查看设备、系统、CPU、内存、存储和当前硬件能力 |
| 巡测 | 配置任务、开始或停止采集、查看实时监控并添加现场事件 |
| 记录 | 管理历次巡测，导出、生成报告或删除指定记录 |
| 传感器 | 查看设备传感器清单，并选择需要记录的采集通道 |

## Root 增强

普通手机无需 Root 也能使用核心采集功能。

如果在设置中主动启用 Root 增强，ComforTroll 可以尝试读取更多设备状态，并配合 **ComforTroll Guard** 伴生 Magisk 模块改善长期采集时的后台可靠性。

Root 权限下可以实现更多功能。但是，代价是什么呢……通常是你终于拥有了整台手机，同时也失去了把问题怪给厂商的权利。

Root 相关操作均为可选项。不同 ROM、内核和厂商实现可能返回不同结果，App 不会把不存在的数据编出来安慰你。

## 安装

从 [Releases](https://github.com/PanGuanWoody/ComforTroll/releases) 下载最新 APK，然后在 Android 设备上安装。

最低支持 Android 6.0（API 23），建议在正式巡测前完成一次锁屏连续采集测试。毕竟“通知栏还在”与“数据真的还在写”之间，隔着一整套安卓后台管理哲学。

## 数据说明

ComforTroll 保存实际收到的传感器事件和时间戳，不会为了让表格好看而伪造严格等间隔数据。

手机传感器适合作为现场辅助观测与状态代理，但不自动等于科研级仪器：

- 环境光传感器不是专业照度计；
- 电池温度和设备温度不是空气温度；
- GNSS 坐标永远没有地图上的小圆点看起来那么自信；
- Root 能增加读取范围，不能凭空焊上一颗新传感器。

请保留精度、质量标记与原始数据。删掉异常值很容易，重新回现场走一遍通常比较痛苦。

## 项目用途

本项目用于环境设计户外数据采集方法探索，可服务于热舒适巡测、田野调查、空间环境调查及相关教学研究。它是一件科研工具，不是医疗设备，也不会因为图标里有巨魔就自动获得北欧神力。
